# 晓辉博士：OpenClaw是捷径但底层逻辑是必修课

> 2026-02-12 · 来源：抖音 @晓辉博士
> 视频ID：7618241003917774245

## 核心观点

OpenClaw 破圈让更多人认识了 AI Agent，但很多人跳过了理解 AI 执行原理的过程，直接享受结果。**这不是理想的学习路径。**

## 关键摘要

1. **不要跳过底层** — OpenClaw 是黑盒，跳过去你不知道 AI 怎么完成任务、质量标准是什么、怎么调优
2. **先从编程工具开始** — 用 Claude Code / Cursor 等"人在环中"的工具，看到 AI 一步步执行的过程
3. **经验要沉淀** — 每次对话后让 AI 沉淀经验（写 skill/SOP），下次不用重复教
4. **Gary Tan 的 Agent 团队** — 一个指挥多个专业 Agent（CEO/工程师/设计师/QA），但自己写 SOP 的过程才是真正的 AI 素养提升
5. **AI 素养 = 投入时间** — 越自主探索，反馈越多，思维进化越快

## 金句

- "光养虾是不够的，还是要从底层原理、工作流实现、经验沉淀和 Agent 能力编排一步步进阶"
- "让 AI 做的事情你必须要自己会做，才能保证你控制住节奏"
- "用 AI 用得好的人都是加班比较多的人"（投入大量时间打磨）
- "以后的工作变成架构师和审美，你是 owner，负责为 AI 结果负责"

## 对我们的启示

- ✅ 我们已经在做的：用 Claude Code + Superpowers + gstack 打磨工作流
- ✅ 经验沉淀：创建 SKILL.md（如 gh-tool-review）
- ⚠️ 需要加强：每次完成任务后主动沉淀经验，而不是一次性交付
- ⚠️ Gary Tan 的 gstack 已装，要实战用起来而不是当摆设

哈喽，下班了。今天我想聊一聊到底该怎么样去跟AI一起进化，或者说以一种什么样的思维模式去指导AI，让它帮你完成一些工作。

那我们知道最近OpenClaw非常火，破圈对吧？很多人都了解到可以养一只龙虾，让龙虾帮你干很多活。但我自己感觉就是龙虾这次的破圈好处是抬高了社会对AI的认知，知道Agent是可以帮你完成很多工作的。但是不好的地方我觉得也非常明显，就是很多人其实直接跳过了去研究和观察AI Agent到底是怎么完成任务的过程，直接来到了享受它结果的状态，我自己觉得不是特别理想，因为在这个过程中我们对AI使用的能力并没有提升。

一个理想的方式是什么呢？我自己觉得还是应该用 Claude Code、Cursor 这样的一些工具，你需要人在环中去看到AI是如何一步一步把这些任务完成的，同时你也能看到它存在哪些漏洞，可以给哪些输入，然后这个闭环是怎么建立起来的。这个任务假设完成的很好，它的经验是如何沉淀下来变成一个skill。这个过程如果被跳过去，那你再去用AI指导OpenClaw干活的时候，一旦你发现它交付的质量不够好，你都不知道该怎么去调它。

我明显感觉到那些用AI用的非常好的人都是投入了非常多时间去跟AI打磨，去看它的产出，去看它的错误，知道它的限制，包括上下文限制、能力限制、质量限制，它什么地方超过你，什么地方不如你。

为什么这里面是有思维认知的变化呢？首先在工作经验可沉淀可复用这一点上，程序员比非程序员有更好的认知，他们知道什么样的工作可以写代码去完成就不用一遍一遍重复做。但非程序员背景的以前重复性脑力活动非常多。所以你在跟AI打磨的过程中，你知道每次对话应该告诉它要沉淀经验，下次就不用重复告诉它该怎么做了。

然后Gary Tan（Y Combinator掌门人）开源了他的一系列agent，相当于一个人指挥一个团队，每个角色都是领域专家。CEO角色评估决策是否正确，资深工程师、设计师、测试。他们不光完成一个任务，还要达到标准和交付质量。你怎么把这些agent写出来？固然可以用开源项目直接用，但你自己写SOP的过程本身就是AI素养的提升。

所以AI这件事要跟它一起进化，需要投入大量时间和精力。越是自主性强、愿意探索的，得到的反馈互动和思维进化就越多。以后大家的工作变成架构师和审美，你是owner，负责为AI结果负责。让AI做的事情你必须要自己会做，才能保证控制住节奏。

AI是全新的物种，需要重新认识它。光养虾是不够的，要从底层原理、工作流实现、经验沉淀和Agent能力编排一步步进阶。这个过程没有捷径，得投入时间投入token去试去学。前沿探索者可能需要四五小时，你现在可能一两个小时就能做得很好，经验可以沉淀，但过程免不了。
