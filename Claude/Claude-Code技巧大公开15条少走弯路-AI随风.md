# Claude Code技巧大公开，15条让你少走弯路

> 来源：抖音 @AI随风 | 详细介绍了十五个 Claude Code 常用技巧

## 15条技巧

1. **设置 cc 别名** — 快速启动 Claude Code
2. **用 `/init` 命令重构/初始化 CLAUDE.md 文档** — 项目规范文件管理
3. **不喜欢终端但又想使用 Claude Code，使用其他 UI** — 不绑定终端
4. **按 Esc 键是停止，按 Esc+Esc 可以直接进入 /rewind 模式进行回滚** — 操作回退
5. **在跟 Claude 对话时，提供详细的反馈链路** — 提高回答质量
6. **安装一个针对你语言的代码智能插件（LSP）** — 提升代码理解能力
7. **尽可能多的使用技能/创造技能** — 发挥 Skill 系统
8. **当你不确定如何处理某件事时，使用计划模式** — `/plan` 先规划再执行
9. **别用自己的话转述 bug，直接把报错贴给 Claude** — 减少信息失真
10. **开启无关的任务时，进行 `/clear` 操作清空上下文** — 避免上下文污染
11. **使用子代理保持主上下文的清晰** — 复杂任务拆分
12. **使用 `/btw` 来进行快速提问** — 不影响主流程的快速查询
13. **使用 `--worktree` 并行开发** — 多分支同时工作
14. **选择合适的 MCP 来提升效率** — 外部工具集成
15. **告诉 Claude 具体要看哪些文件** — 缩小搜索范围

## 核心观点

- Claude Code 的核心是**上下文管理**：`/clear`、子代理、`/btw` 都是为了保持上下文干净
- **Skill 系统**是 Claude Code 的最大差异化能力，要充分利用
- **报错直接贴**比口头描述准确得多
