# MCO 功能模块

> 本文档详细说明 MCO 的各功能模块

---

## 模块概览

```
mco
├── review      # 结构化代码审查
├── run         # 通用任务执行
├── doctor      # 环境健康检查
└── agent       # Agent 注册表管理
    ├── list    # 列出所有 Agent
    └── check   # 检查单个 Agent
```

---

## 1. Review 模块

### 功能描述

结构化代码审查，输出标准化的 findings（含严重级别、分类、证据、建议）。

### 基本用法

```bash
mco review \
  --repo . \
  --prompt "Review for security vulnerabilities and performance issues." \
  --providers claude,codex,gemini,opencode,qwen \
  --json
```

### 输出格式

| 格式 | 参数 | 用途 |
|------|------|------|
| 人类可读报告 | `--format report`（默认） | 终端阅读 |
| PR Markdown | `--format markdown-pr` | 作为 GitHub PR 评论发布 |
| SARIF 2.1.0 | `--format sarif` | 上传到 GitHub Code Scanning |
| 机器可读 JSON | `--json` | 下游自动化 |

### Finding 结构

每个 finding 包含：

```json
{
  "id": "unique-finding-id",
  "severity": "high|medium|low|info",
  "category": "security|performance|correctness|maintainability",
  "title": "简短描述",
  "description": "详细说明",
  "evidence": "代码证据",
  "recommendation": "修复建议",
  "file_path": "文件路径",
  "line_number": 123,
  "detected_by": ["claude", "codex"],
  "agreement_ratio": 0.6,
  "consensus_score": 0.72,
  "consensus_level": "confirmed"
}
```

### Review 协作模式

| 模式 | 参数 | 行为 |
|------|------|------|
| **并行** | 默认 | 所有 provider 审查同一份范围 |
| **Chain** | `--chain` | 顺序执行，后一个 provider 看到前一个的分析 |
| **Debate** | `--debate` | 对合并后的 finding 再做一轮 challenge/refine |
| **文件分工** | `--divide files` | 按文件切片分工 |
| **维度分工** | `--divide dimensions` | 每个 provider 负责不同审查维度 |

## 2. Run 模块

### 功能描述

通用多 Agent 任务执行，不强制输出格式，provider 自由完成任务。

### 基本用法

```bash
mco run \
  --repo . \
  --prompt "Summarize the architecture of this project." \
  --providers claude,codex \
  --json
```

### 适用场景

| 场景 | 示例 Prompt |
|------|------------|
| 架构分析 | "Summarize the architecture of this project." |
| 文档生成 | "Generate API documentation for the endpoints." |
| 代码理解 | "Explain how the authentication flow works." |
| 重构建议 | "Suggest refactoring opportunities for better testability." |

## 3. Doctor 模块

### 功能描述

在执行任务前检查所有 Agent 的安装、可达性和认证状态。

### 基本用法

```bash
# 人类可读输出
mco doctor

# JSON 格式输出
mco doctor --json
```

### 检查内容

| 检查项 | 说明 |
|--------|------|
| 二进制存在 | CLI 命令是否可执行 |
| 版本信息 | 当前安装的版本 |
| 认证状态 | API Key 是否有效 |

### 输出示例

```
Provider     Status    Version    Auth
─────────────────────────────────────
claude       ✓ OK      1.0.0      ✓
codex        ✓ OK      0.5.0      ✓
gemini       ✗ Missing -          -
opencode     ✓ OK      1.2.0      ✓
qwen         ✓ OK      0.1.0      ✓
```

## 4. Agent 模块

### 功能描述

管理内置和自定义 Agent 注册表。

### 子命令

| 命令 | 说明 |
|------|------|
| `mco agent list` | 列出所有可见的 Agent（内置 + 自定义） |
| `mco agent check <name>` | 检查某个自定义 Agent 或 Ollama 包装器 |

### 基本用法

```bash
# 列出所有 Agent
mco agent list

# 检查特定 Agent
mco agent check my-ollama
```

## 5. 实时流模式

### 功能描述

提供实时的执行进度反馈。

### 流模式选项

| 模式 | 参数 | 用途 |
|------|------|------|
| JSONL 流 | `--stream jsonl` | 机器可读事件流 |
| 实时终端 | `--stream live` | 交互式终端实时进度视图 |

### 基本用法

```bash
# JSONL 流模式
mco review --repo . --prompt "Review code" --stream jsonl

# 实时终端模式
mco review --repo . --prompt "Review code" --stream live
```

## 6. 路径约束模块

### 功能描述

限制 Agent 可访问的文件范围。

### 参数说明

| 参数 | 说明 |
|------|------|
| `--allow-paths` | 允许访问的路径列表 |
| `--target-paths` | 目标审查路径 |
| `--enforcement-mode` | 执行模式：`strict` 或 `permissive` |

### 基本用法

```bash
mco run \
  --repo . \
  --prompt "Analyze the adapter layer." \
  --providers claude,codex \
  --allow-paths runtime,scripts \
  --target-paths runtime/adapters \
  --enforcement-mode strict
```

## 7. Diff 审查模块

### 功能描述

仅审查代码变更部分。

### 参数说明

| 参数 | 说明 |
|------|------|
| `--diff` | 仅审查相对主分支 merge-base 的变更 |
| `--staged` | 仅审查 git staged 变更 |
| `--unstaged` | 仅审查未暂存变更 |
| `--diff-base` | 分支 diff 的 Git 基线 |

### 基本用法

```bash
# 审查相对于主分支的变更
mco review --repo . --prompt "Review changes" --diff

# 审查 staged 变更
mco review --repo . --prompt "Review staged changes" --staged

# 审查最近 3 次提交的变更
mco review --repo . --prompt "Review changes" --diff-base HEAD~3
```

## 8. Token 用量追踪

### 功能描述

追踪各 Agent 和汇总的 token 消耗。

### 基本用法

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --include-token-usage
```

### 输出示例

```json
{
  "token_usage": {
    "claude": { "input": 1500, "output": 800 },
    "codex": { "input": 1200, "output": 600 },
    "total": { "input": 2700, "output": 1400 }
  }
}
```

## 9. 共识总结模块

### 功能描述

额外执行一轮 LLM 总结，输出跨 Agent 的共识/分歧摘要。

### 基本用法

```bash
mco review \
  --repo . \
  --prompt "Review for bugs." \
  --synthesize \
  --synth-provider claude
```

### 输出内容

- **共识点**：所有 Agent 一致认同的问题
- **分歧点**：Agent 之间存在不同看法的地方
- **建议行动**：下一步应该做什么

## 10. 辩论模式

### 功能描述

让多个 Agent 在最终输出前对合并后的 finding 再做一轮质疑或补充。

### 基本用法

```bash
mco review \
  --repo . \
  --prompt "审查这个 PR，并在最终排序前质疑证据不足的 finding。" \
  --providers claude,codex,gemini \
  --debate
```

### 辩论流程

1. 各 Agent 独立审查，输出 findings
2. 合并 findings，进行共识分析
3. 将合并结果发回各 Agent 进行挑战/补充
4. 最终输出经过辩论的 findings

## 11. 分工模式

### 功能描述

支持按文件或按审查维度拆分任务。

### 文件分工 (`--divide files`)

```bash
mco review \
  --repo . \
  --prompt "审查这个 PR 的正确性和性能问题。" \
  --providers claude,codex,gemini \
  --divide files
```

- 按文件大小优先、均衡分配到不同 provider
- 大文件优先后轮转分配

### 维度分工 (`--divide dimensions`)

```bash
mco review \
  --repo . \
  --prompt "全面审查这个 PR。" \
  --providers claude,codex,gemini \
  --divide dimensions
```

五个审查维度：
1. **安全性** (Security)
2. **性能** (Performance)
3. **可维护性** (Maintainability)
4. **正确性** (Correctness)
5. **错误处理** (Error Handling)

> 注意：`--divide` 与 `--chain`、`--debate` 互斥
