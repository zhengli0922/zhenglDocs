# MCO 高级特性

> 本文档详细说明 MCO 的高级特性

---

## 自定义 Agent

### 概述

MCO 支持注册自定义 Agent，包括：
- ACP (Agent Client Protocol) Agent
- Shim Agent（基于命令行）
- Ollama 本地模型

### 配置文件

配置文件按以下优先级加载：

1. `.mco/agents.yaml`（项目级，最高优先级）
2. `.mcorc.yaml`（项目根目录）
3. `~/.mco/agents.yaml`（全局）

### 配置示例

```yaml
# .mco/agents.yaml
agents:
  # 1. ACP Agent（通过 JSON-RPC 协议）
  - name: my-acp-agent
    transport: acp
    command: my-agent --acp
    permission_keys: [sandbox]

  # 2. Shim Agent（基于命令行输出解析）
  - name: my-shim-agent
    transport: shim
    command: my-review-bot --json

  # 3. Ollama 本地模型
  - name: my-ollama
    model: qwen2.5-coder:14b

  # 4. 另一个 Ollama 模型
  - name: deepseek-coder
    model: deepseek-coder:6.7b
```

### Agent 类型说明

#### ACP Agent

使用 Agent Client Protocol (JSON-RPC) 通信：

```yaml
- name: my-acp-agent
  transport: acp
  command: my-agent --acp
  permission_keys: [sandbox]
```

特点：
- 标准化协议
- 支持双向通信
- 适合复杂 Agent

#### Shim Agent

基于命令行输出解析：

```yaml
- name: my-shim-agent
  transport: shim
  command: my-review-bot --json
```

特点：
- 简单易用
- 适合现有工具包装
- 只需输出 JSON 格式

#### Ollama 模型

自动包装本地 Ollama 模型：

```yaml
- name: my-ollama
  model: qwen2.5-coder:14b
```

特点：
- 无需额外开发
- 支持本地推理
- 隐私友好

### 管理 Agent

```bash
# 列出所有 Agent（内置 + 自定义）
mco agent list

# 检查特定 Agent
mco agent check my-ollama
```

### 使用自定义 Agent

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --providers claude,my-ollama,deepseek-coder
```

## 临时自定义 Agent

### 使用 `--agent` 参数

无需配置文件，直接指定临时 Agent：

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --agent my-agent "python my_reviewer.py --json"
```

### 与 Transport 结合

```bash
# Shim transport
mco review \
  --repo . \
  --prompt "Review code" \
  --transport shim \
  --agent my-agent "python my_reviewer.py --json"

# ACP transport
mco review \
  --repo . \
  --prompt "Review code" \
  --transport acp \
  --agent my-agent "python my_acp_agent.py"
```

## 辩论模式 (`--debate`)

### 概述

让多个 Agent 在最终输出前对合并后的 finding 再做一轮质疑或补充。

### 工作流程

```
┌─────────────────────────────────────────────────────────┐
│                    第一轮：独立审查                       │
├─────────────────────────────────────────────────────────┤
│  Claude ──→ Findings A                                  │
│  Codex  ──→ Findings B                                  │
│  Gemini ──→ Findings C                                  │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                    合并与共识分析                         │
├─────────────────────────────────────────────────────────┤
│  合并 A + B + C → Findings M                            │
│  计算共识分数                                            │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                    第二轮：辩论                          │
├─────────────────────────────────────────────────────────┤
│  Claude 审查 M → 质疑/补充                              │
│  Codex  审查 M → 质疑/补充                              │
│  Gemini 审查 M → 质疑/补充                              │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                    最终输出                              │
├─────────────────────────────────────────────────────────┤
│  经过辩论验证的 Findings                                 │
└─────────────────────────────────────────────────────────┘
```

### 使用示例

```bash
mco review \
  --repo . \
  --prompt "审查这个 PR，并在最终排序前质疑证据不足的 finding。" \
  --providers claude,codex,gemini \
  --debate
```

### 适用场景

- 高风险代码变更
- 安全审查
- 关键业务逻辑审查
- 需要 high confidence 的场景

## 分工模式 (`--divide`)

### 概述

支持按文件或按审查维度拆分任务。

### 文件分工 (`--divide files`)

#### 工作方式

```
文件列表：
- large_file.py (5000 行)
- medium_file.py (2000 行)
- small_file.py (500 行)
- tiny_file.py (100 行)

分配策略：
- Claude  → large_file.py + tiny_file.py
- Codex   → medium_file.py
- Gemini  → small_file.py
```

#### 使用示例

```bash
mco review \
  --repo . \
  --prompt "审查这个 PR 的正确性和性能问题。" \
  --providers claude,codex,gemini \
  --divide files
```

#### 特点

- 大文件优先后轮转分配
- 均衡负载
- 适合大型 PR 审查

### 维度分工 (`--divide dimensions`)

#### 五个审查维度

| 维度 | 说明 |
|------|------|
| **安全性** (Security) | SQL 注入、XSS、认证问题等 |
| **性能** (Performance) | N+1 查询、内存泄漏、算法效率等 |
| **可维护性** (Maintainability) | 代码结构、命名、注释等 |
| **正确性** (Correctness) | 逻辑错误、边界条件等 |
| **错误处理** (Error Handling) | 异常处理、错误恢复等 |

#### 工作方式

```
Provider 分配：
- Claude  → 安全性 (Security)
- Codex   → 性能 (Performance)
- Gemini  → 可维护性 (Maintainability)
```

#### 使用示例

```bash
mco review \
  --repo . \
  --prompt "全面审查这个 PR。" \
  --providers claude,codex,gemini \
  --divide dimensions
```

#### 特点

- 每个 Provider 聚焦一个维度
- 避免重复工作
- 更深入的专项审查

### 互斥关系

> 注意：`--divide` 与 `--chain`、`--debate` 互斥

## Chain 模式 (`--chain`)

### 概述

顺序执行 Provider，后一个 Provider 看到前一个的分析。

### 工作流程

```
┌─────────────────────────────────────────────────────────┐
│                    Step 1: Claude 执行                   │
│                    输出: Analysis A                      │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                    Step 2: Codex 执行                    │
│                    输入: Analysis A                      │
│                    输出: Analysis B                      │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│                    Step 3: Gemini 执行                   │
│                    输入: Analysis B                      │
│                    输出: Final Analysis                  │
└─────────────────────────────────────────────────────────┘
```

### 使用示例

```bash
mco run \
  --repo . \
  --prompt "Analyze the architecture and suggest improvements." \
  --providers claude,codex,gemini \
  --chain
```

### 适用场景

- 架构分析
- 复杂问题诊断
- 需要逐步深入的场景
- Agent 之间需要协作的场景

## 实时流模式 (`--stream`)

### JSONL 流 (`--stream jsonl`)

机器可读的事件流：

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --stream jsonl
```

输出示例：
```jsonl
{"event": "start", "provider": "claude"}
{"event": "progress", "provider": "claude", "bytes": 1024}
{"event": "complete", "provider": "claude", "findings": 5}
{"event": "start", "provider": "codex"}
{"event": "complete", "provider": "codex", "findings": 3}
{"event": "merge", "total_findings": 6}
{"event": "done"}
```

### 实时终端 (`--stream live`)

交互式终端实时进度视图：

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --stream live
```

输出示例：
```
MCO Review Progress
===================

Claude    ████████████████████ Complete (5 findings)
Codex     ████████████░░░░░░░░ In progress...
Gemini    ░░░░░░░░░░░░░░░░░░░░ Waiting

Total: 5 findings so far
```

## Transport 模式

### Shim Transport（默认）

通过 stdout 解析通信：

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --transport shim
```

特点：
- 简单直接
- 适合大多数场景
- 基于命令行输出解析

### ACP Transport

通过 Agent Client Protocol (JSON-RPC) 通信：

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --transport acp
```

特点：
- 标准化协议
- 支持双向通信
- 适合复杂 Agent

## 从文件读取 Prompt

### 使用 `--file` 参数

```bash
# 从文件读取
mco review \
  --repo . \
  --file prompt.txt

# 从 stdin 读取
echo "Review code" | mco review --repo . --file -
```

### 适用场景

- 长提示词
- 复杂的审查指令
- 脚本化工作流

---

## 静默模式 (`--quiet`)

仅输出最终文本，不带标题和格式：

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --quiet
```

> 注意：与 `--json`/`--stream` 互斥

## 完整高级示例

### 安全审查 + 自定义 Agent + 辩论

```bash
mco review \
  --repo . \
  --prompt "全面安全审查" \
  --providers claude,codex,gemini,my-ollama \
  --debate \
  --synthesize \
  --format sarif \
  --save-artifacts \
  --stream live
```

### 维度分工 + Chain + 共识总结

```bash
# 注意：--divide 和 --chain 互斥，这里用 --divide dimensions
mco review \
  --repo . \
  --prompt "全面代码质量审查" \
  --providers claude,codex,gemini \
  --divide dimensions \
  --synthesize \
  --format markdown-pr
```

### 实时流 + 自定义 Agent

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --agent my-reviewer "python my_reviewer.py --json" \
  --providers claude,codex \
  --transport shim \
  --stream live
```
