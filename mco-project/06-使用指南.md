# MCO 使用指南

> 本文档提供 MCO 的安装、配置与使用示例

---

## 安装

### 方式一：npm 安装（推荐）

```bash
npm i -g @tt-a1i/mco
```

> 需要 Python 3 已在系统 PATH 中

### 方式二：从源码安装

```bash
git clone https://github.com/mco-org/mco.git
cd mco
python3 -m pip install -e .
```

### 方式三：pip 安装

```bash
pip install mco
```

## 快速开始

### 1. 环境检查

```bash
mco doctor
```

输出示例：
```
Provider     Status    Version    Auth
─────────────────────────────────────
claude       ✓ OK      1.0.0      ✓
codex        ✓ OK      0.5.0      ✓
gemini       ✗ Missing -          -
```

### 2. 第一次审查

```bash
mco review \
  --repo . \
  --prompt "Review this repository for high-risk bugs and security issues." \
  --providers claude,codex,qwen
```

## 常用场景

### 场景 1：PR 代码审查

```bash
# 审查当前分支相对于主分支的变更
mco review \
  --repo . \
  --prompt "Review this PR for bugs, security issues, and code quality." \
  --providers claude,codex,gemini \
  --diff \
  --format markdown-pr \
  --synthesize
```

### 场景 2：CI/CD 安全扫描

```yaml
# .github/workflows/security-review.yml
name: Security Review

on: [push, pull_request]

jobs:
  security-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup MCO
        run: npm i -g @tt-a1i/mco

      - name: Run Security Review
        run: |
          mco review \
            --repo . \
            --prompt "Review for security vulnerabilities" \
            --providers claude,codex \
            --format sarif \
            --save-artifacts

      - name: Upload SARIF
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: reports/review/*/findings.sarif
```

### 场景 3：架构分析

```bash
mco run \
  --repo . \
  --prompt "Analyze the architecture of this project. Identify design patterns, potential issues, and improvement opportunities." \
  --providers claude,gemini,qwen \
  --json
```

### 场景 4：部署前健康检查

```bash
# 检查所有 Provider 状态
mco doctor --json > doctor-report.json

# 如果需要，解析 JSON 检查是否有失败的 Provider
```

### 场景 5：辩论模式深度审查

```bash
mco review \
  --repo . \
  --prompt "审查这个 PR，并在最终排序前质疑证据不足的 finding。" \
  --providers claude,codex,gemini \
  --debate \
  --synthesize
```

### 场景 6：维度分工审查

```bash
mco review \
  --repo . \
  --prompt "全面审查代码质量。" \
  --providers claude,codex,gemini \
  --divide dimensions
```

维度分工自动分配：
- Claude → 安全性 (Security)
- Codex → 性能 (Performance)
- Gemini → 可维护性 (Maintainability)

## 配置

### 自定义 Agent 配置

配置文件按以下优先级加载：

1. `.mco/agents.yaml`（项目级）
2. `.mcorc.yaml`（项目根目录）
3. `~/.mco/agents.yaml`（全局）

#### 示例配置

```yaml
# .mco/agents.yaml
agents:
  # ACP Agent
  - name: my-acp-agent
    transport: acp
    command: my-agent --acp
    permission_keys: [sandbox]

  # Shim Agent（基于命令行）
  - name: my-shim-agent
    transport: shim
    command: my-review-bot --json

  # Ollama 本地模型
  - name: my-ollama
    model: qwen2.5-coder:14b
```

#### 查看配置的 Agent

```bash
# 列出所有 Agent
mco agent list

# 检查特定 Agent
mco agent check my-ollama
```

### Provider 权限配置

```bash
mco review \
  --repo . \
  --prompt "Review code" \
  --providers claude,codex \
  --provider-permissions-json '{
    "claude": {"permission_mode": "acceptEdits"},
    "codex": {"sandbox": "workspace-write"}
  }'
```

## 产物文件

启用 `--save-artifacts` 后生成的文件：

```
reports/review/<task_id>/
  ├── summary.md          # 人类可读摘要
  ├── decision.md         # PASS / FAIL / ESCALATE / PARTIAL
  ├── findings.json       # 聚合后的标准化 findings
  ├── run.json            # 机器可读执行元数据
  ├── providers/          # 各 provider 结果 JSON
  │   ├── claude.json
  │   ├── codex.json
  │   └── gemini.json
  └── raw/                # 原始 stdout/stderr 日志
      ├── claude.log
      ├── codex.log
      └── gemini.log
```

## 最佳实践

### 1. 选择合适的 Provider 组合

| 场景 | 推荐组合 |
|------|----------|
| 快速审查 | claude, codex |
| 全面审查 | claude, codex, gemini, qwen |
| 安全重点 | claude, codex, gemini |
| 性能重点 | gemini, qwen |
| 中文项目 | qwen + claude |

### 2. 使用合适的输出格式

| 场景 | 格式 |
|------|------|
| 终端查看 | `--format report`（默认） |
| PR 评论 | `--format markdown-pr` |
| CI/CD 集成 | `--format sarif` |
| 自动化处理 | `--json` |

### 3. 利用共识机制

- 使用 `--synthesize` 获取共识总结
- 关注 `confirmed` 级别的 findings
- 对 `needs-verification` 的 findings 进行人工复核

### 4. 合理设置超时

```bash
# 对于大型仓库，增加超时时间
mco review \
  --repo . \
  --prompt "Review code" \
  --providers claude,codex,gemini \
  --stall-timeout 1800 \
  --review-hard-timeout 3600
```

### 5. 使用 Diff 模式提高效率

```bash
# 只审查变更部分，节省时间
mco review \
  --repo . \
  --prompt "Review changes" \
  --diff \
  --providers claude,codex
```

## 故障排除

### Provider 未找到

```bash
# 检查 Provider 状态
mco doctor

# 确保 CLI 在 PATH 中
which claude
which codex
```

### 认证失败

```bash
# 检查认证状态
mco doctor

# 重新认证
claude auth login
```

### 超时问题

```bash
# 增加超时时间
mco review \
  --repo . \
  --prompt "Review code" \
  --stall-timeout 1800 \
  --review-hard-timeout 3600
```

### 内存问题

```bash
# 限制并行度
mco review \
  --repo . \
  --prompt "Review code" \
  --max-provider-parallelism 2
```
