# Skill: 知识库整理助手

> 根据 README.md 规则，自动将新笔记分类到对应目录

---

## 触发场景

- 用户新建笔记或询问"这个笔记放哪里"
- 用户要求整理/归档笔记
- 用户提供了笔记内容需要分类存储

---

## 分类决策流程

### 第一步：识别内容类型

检查笔记内容中的关键词：

| 类型 | 关键词示例 | 目标目录 |
|------|-----------|---------|
| 股市日记 | 股市、A股、大盘、板块、涨跌、K线、开盘、收盘、成交量 | `/股市日记/` |
| 视频笔记 | 抖音、B站、YouTube、视频、直播、v.douyin、bilibili | `/Videos/` |
| 项目笔记 | 项目、功能开发、需求、迭代、截止日期 | `/Projects/` |
| 领域笔记 | 编程、理财、工作、技能、长期、持续维护 | `/02-Areas/` |
| 资源笔记 | 来源、参考、文章、博客、GitHub、整理自 | `/03-Resources/` |

### 第二步：确定目标目录

```
内容类型判断流程：

1. 包含股市/交易数据？
   └─ 是 → /股市日记/YYYY-MM-DD-主题.md

2. 来源是视频/直播？
   └─ 是 → /Videos/主题-来源.md

3. 有明确项目目标和截止日期？
   └─ 是 → /Projects/项目名-描述.md

4. 持续维护的主题？
   └─ 是 → /02-Areas/领域/主题.md

5. 外部参考资料？
   └─ 是 → /03-Resources/资源名.md
```

### 第三步：生成文件名

| 目录 | 命名格式 | 示例 |
|------|---------|------|
| 股市日记 | `YYYY-MM-DD-主题.md` | `2026-03-24-开盘总结.md` |
| Videos | `主题-来源.md` | `OpenClaw自定义AI人格-小龙虾.md` |
| Projects | `项目名-描述.md` | `OpenClaw-AI人格自定义.md` |
| Areas | `领域/主题.md` | `编程/OpenClaw养虾省钱指南.md` |
| Resources | `资源名.md` | `Claude-Code最佳实践完全指南.md` |

## Frontmatter 模板

### 股市日记
```yaml
---
title: YYYY-MM-DD A股XXX总结
date: YYYY-MM-DD
tags: [A股, 开盘总结, 上证指数, 深证成指, 创业板指]
type: daily-stock
---
```

### 视频笔记
```yaml
---
title: 标题
date: YYYY-MM-DD
source: 视频链接
author: 作者
type: video
tags: [标签1, 标签2]
---
```

### 资源笔记
```yaml
---
title: 标题
source: 来源链接
date: YYYY-MM-DD
tags: [标签1, 标签2]
---
```

## 执行步骤

当用户需要整理笔记时：

1. **分析内容**：识别笔记主题和关键词
2. **确定类型**：根据决策流程确定分类
3. **生成文件名**：按命名规范生成文件名
4. **添加 Frontmatter**：根据类型添加对应的元数据
5. **写入文件**：将内容写入目标目录
6. **确认结果**：告知用户文件存放位置

---

## 示例对话

**用户**：帮我整理一个关于 OpenClaw 配置的视频笔记

**Claude**：
1. 识别到来源是视频 → 分类为视频笔记
2. 生成文件名：`OpenClaw配置指南-小龙虾.md`
3. 目标目录：`/Videos/`
4. 创建文件并添加 frontmatter
5. 确认：已保存到 `Videos/OpenClaw配置指南-小龙虾.md`

---

## 注意事项

- 文件名使用中文，保持可读性
- 日期格式统一使用 `YYYY-MM-DD`
- 标签使用小写英文或中文，避免特殊字符
- 如果内容跨多个分类，优先选择更具体的分类
- 股市日记必须是当天的记录，历史数据按实际日期命名
