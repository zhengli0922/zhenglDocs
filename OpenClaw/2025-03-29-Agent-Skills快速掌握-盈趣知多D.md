# 10分钟快速掌握Agent Skills

> 来源：[盈趣知多D - 今日头条](https://m.toutiao.com/is/qT-D_nYeb5M/)

## 一、交互方式演进（5个Level）

### Level 1：Prompt 提示词
- 普通提示词 → 结果不可控
- **结构化 Prompt** → 设定角色(Roll)、任务(Task)、规则(Rules)，结果规矩多了

### Level 2：Commands 命令
- 结构化提示词太长，每次复制粘贴太反人类
- 解决方案：封装成文件，用快捷指令（如 `/resume`）一键调用
- **本质：把最佳实践固化成快捷键**

### Level 3：System Prompt 系统提示词
- 长对话中 AI 容易忘记初始规则（上下文丢失）
- 用 Cursor Rules 或 `.clinerules` 等文件，系统提示词有**最高优先级**
- 不管用户怎么聊，AI 始终受系统级规则约束

### Level 4：Metadata 原数据
- 100种任务不可能全塞进上下文 → 给文件贴智能标签(tag)
- **按需加载**：只把少量 metadata 发给模型，模型判断需要哪个再拉取
- 系统扩展能力彻底打开，不用抠 token 上限

### Level 5：References & Scripts 参考资料与脚本
- AI 不再一把抓，通过**渐进式披露**扩展能力
- References：按需加载产品手册、工程文档
- Scripts：直接调脚本生成 PDF、跑数据分析
- **AI 不仅能动嘴，还能动手了**

## 二、Agent Skills 的本质

**官方定义**：被封装的动态能力单元

一个 Skill 包含 4 样东西：

| 组成 | 比喻 | 说明 |
|------|------|------|
| `SKILL.md`（系统提示词） | 岗位说明书 | 规定角色、法则、语气 |
| Metadata（路由标签） | 门牌号 | AI 据此判断何时激活哪个技能 |
| References（数据素材） | 参考资料库 | 业务白皮书、模板文件 |
| Scripts（执行工具） | 干活工具箱 | Python脚本、代码生成 |

**公式**：Skill = 岗位说明书 + 门牌号标签 + 参考资料库 + 干活工具箱

## 三、Skills 工作流程（5步）

1. **用户输入**："帮我写简历"
2. **客户端**：收集所有 skill 的 metadata 标签，发给模型
3. **模型选择**：看标签，判断归哪个技能包管
4. **加载详情**：客户端加载选中 skill 的 SKILL.md（岗位说明书）
5. **执行**：读 references、跑 scripts，输出结果

**核心：按需加载，不一股脑全塞给模型**

## 四、核心概念辨析

### Skill vs MCP
| | MCP | Skill |
|--|-----|-------|
| 比喻 | **手**（工具箱） | **大脑**（操作指南+经验） |
| 解决 | 能不能干的**连接**问题 | 干得好不好的**经验**问题 |
| 关系 | **配合关系，不是替代** | |

MCP 给模型配了灵活的手，但未必有经验。Skill 规定遇到什么情况用哪把刀、切多厚。

### Skill vs Workflow
| | Workflow | Skill |
|--|----------|-------|
| 比喻 | **地铁**（固定轨道） | **网约车**（灵活绕路） |
| 特点 | 固定编排，人为锁死 | 模型驱动，动态决策 |
| 缺点 | 遇到新情况容易卡壳 | - |

Workflow 像铺好的铁轨，Skill 像神经元网络，每一步看情况决定下一步。

## 总结

从 Prompt → Commands → System Prompt → Metadata → Agent Skills，是大模型智能化的**质变**：

> 不再是向 AI 发送文字，而是在**配置 AI 的专业能力**。
> 把提示词、业务数据、执行代码封装成 Skill，把通用聊天伴侣变成具备领域经验的专家代理。

第一部分我们会过一遍交互方式的演进。搞懂了这个源头，对AI的理解就不只是停留在表面了。

第四部分绝对是面试和实战中最容易被问到，也是最容易踩坑的地方。我会把MCP、workflow这些词拿出来对比一下，理清这些以后再看到外面出了什么新的框架或者是什么协议啊，我们都能一眼看透本质做到心里有底。

**Level 1 - Prompt**
最左边这个图，普通的prompt就像是你随口跟AI说了一句，帮我写个简历，结果他可能洋洋洒洒的给你写了一堆的废话，排版也是乱七八糟。为了解决这个呢，我们摸索出了结构化prompt，给他设定具体的roll角色，task任务，还有rules规则。加上这种结构化的约束之后，AI给出的东西就规矩多了。

**Level 2 - Commands**
结构化的提示词虽然好用，但是太长了。我们平时干活总不能每次都去复制粘贴几百字的要求进去。解决方案就是把它封装成一个文件，以后只需要敲一个快捷指令就能调出那一整套复杂的规则。这一步就是把我们摸索出来的最佳实践变成了可以随时调用的快捷键，把效率给固化下来了。

**Level 3 - System Prompt**
平时聊天聊着聊着，上下文一多，AI就特别容易断片。这时候就得用cursor rules或者是clinerules这样的文件。system prompt永远在最上面，有一票否决的最高优先级，不管用户怎么聊，AI脑子里那根弦总是绷紧的。

**Level 4 - Metadata**
如果AI助手要处理100种不同的任务，总不能把100个场景的背景资料全塞进脑子里。做法是给每个文件贴上一个智能标签tag，只把少量的metadata发给模型，模型判断需要什么再按需加载。系统的扩展能力一下子就被彻底打开了。

**Level 5 - References & Scripts**
AI的能力通过渐进式披露来扩展。一边是references，遇到不懂的可以按需加载产品手册、工程文档。另一边是scripts，AI不但知道你要干什么，他还能直接调脚本去生成PDF、跑数据分析。它不仅能动嘴，还能真刀真枪的动手了。

**Skill 的4个组成部分**
1. SKILL.md（系统提示词）- 岗位说明书，规定角色、法则、语气
2. Metadata（路由标签）- 门牌号，系统据此判断何时激活
3. References（数据素材）- 参考资料库，业务白皮书、模板
4. Scripts（执行工具）- 干活工具箱，Python脚本

**Skill vs MCP**
MCP是一套标准化的工具接口，相当于给大模型配了一双灵活的手。但是手里有工具未必有经验。Agent skill相当于操作指南和行业经验，规定遇到什么情况该用哪把刀切多厚。MCP解决能不能干的连接问题，skill解决干得好不好的经验问题。大脑和手是配合关系不是替代关系。

**Skill vs Workflow**
Workflow是固定编排的，第一步干什么第二步干什么全是在系统设计时人为锁死的，遇到新情况只能死板按轨道走容易卡壳。Agent skill由模型驱动，执行路径不是写死的，AI每走一步看情况即时决定下一步。Workflow像地铁（固定轨道），skill像网约车（灵活绕路）。
