# OpenClaw记忆升级教程

> 来源：[小代不懂代码 - 抖音](https://v.douyin.com/XZYFnXLOBdo/)

## 核心观点

默认记忆差的原因：
1. **保存时机太晚**
2. **保存内容不够准**，工具输出容易撑爆上下文
3. **默认很难回查原始对话**
4. **压缩上下文时细节被忽略**

默认只有短期记忆和长期记忆两个文件夹，短期记忆装满后开始压缩总结，重要信息来不及写入长期记忆就会丢失。

## 升级方案（四步）

### 1. 理顺记忆文件结构
分层管理记忆文件，不要全堆一个文件夹，方便后续检索时拿到对的信息。

### 2. 保护最近对话
给最近约 2万 token 画优先保护区，防止刚说的任务目标、修正约束被总结掉。
同时按轮次保护：最后四轮对话完整保留不做摘要。

### 3. 改 memory flash
- 解决触发太晚、写入不精准的问题
- 加 `no flash` 约束，避免硬写无意义摘要造成噪音

### 4. 安装 QMD 插件
实现全量索引，支持关键词+语义双重搜索（向量检索），让 AI 能搜索原始对话历史。

## 使用习惯建议
- 重要信息明确告诉 AI 保存到记忆中
- 长会话结束前列出应保存的关键信息
- 每周回顾记忆文件，删除无用信息

给大家推荐一个我自己在使用的提升小龙虾记忆系统的方案，这是前段时间一位国外的技术大佬分享的方法，那我自己用下来感觉不错，所以推荐给大家。

首先我们要弄清楚它默认的记忆力为什么会差，其实就四个原因：

第一保存时机太晚，第二保存内容不够准，而且工具输出很容易会把上下文撑爆，第三默认很难直接回查原始绘画，第四一旦开始压缩上下文很多细节就会被忽略。

那么它的记忆文件夹只有两个，一个是短期记忆，一个是长期记忆，问题就在于短期记忆一旦装满，系统就会开始压缩，也就是把旧内容压缩总结，那如果重要的信息没有来得及写进长期记忆，细节就会容易丢掉。

所以我们要升级重构这套记忆机制，核心目标就是让该写进长期记忆的东西能更早更准的写进去。

好，下面进行具体的配置，第一步我们要先把记忆文件结构给理顺，把这段提示词发给他，让他去创建几个目录。为什么要先做这一步呢？因为默认情况下它会把所有的记忆往一个文件夹里面堆，那这样最大的问题就是后面它在检索的时候重点会越来越不清楚，所以这里要做分层，这样做的好处是后面它在回忆信息时会更容易拿到对的东西。

接下来改第二层，是先保护最近对话，再处理上下文膨胀，把这段提示词发给他。因为小龙虾的失忆很多时候不是完全没有记住过去，而是因为它最近刚说完的内容被压缩了，所以这里要做几层保护，让压缩更保守，把最近大约两万 token 画成一个优先保护区，这样刚刚讲过的任务目标、修正约束不会被总结掉。然后再按轮次加一层保险，最后四轮对话完整保留，不做摘要，也就是说既按 token 保护也按轮次保护。那其他几个参数主要就是给当前的任务留足空间。

接下来我们改 memory flash，默认的 memory flash 有两个问题，第一是触发太晚，第二是写进去的内容也不够精准，把这段提示词发给他，让他去修改。注意这里为什么要加一个 no flash，因为不是每次聊天都值得写进长期记忆。如果没有这个约束，它很可能为了完成任务就会硬写一堆没有意义的摘要，那时间一长日志里全都是噪音，就是垃圾太多了。

OK，到这一步，其实结构化的记忆已经比默认状态好很多了。但是我们还需要安装一个最强的引擎，就是 QMD，你直接把这段指令发给他。那通过安装和配置 QMD 插件，就能实现全量索引，让 AI 能够对原始对话历史进行关键词加语义的双重搜索，也就是向量检索。

最后还要给它加一个规则约束的指令，把这段发给他。那改造完这整套记忆结构，我们自己平时也要养成个好习惯啊，如果是重要的信息你就要告诉它保存到记忆中。在每一次长会话结束的时候，你让它给列出来应该保存的关键信息。然后每周回顾一次记忆文件，要删掉一些没有用的信息。OK，做完以上这些步骤，小龙虾的长期使用体验就会得到巨大的提升。
