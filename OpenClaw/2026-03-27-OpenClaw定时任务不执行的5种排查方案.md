# OpenClaw 定时任务不执行的5种排查方案

> 作者：磊哥聊AI
> 来源：抖音 @磊哥聊AI
> 关键词：OpenClaw、定时任务、Cron、Gateway

## 定时任务分类

1. **一次性任务** — 执行一次就没了（如：3分钟后提醒喝水）
2. **重复性任务**
   - 固定间隔（如：每30分钟抓取邮件）
   - 定点执行（如：每天9:30抓取新闻）

## 5种排查思路

### 1. 确认任务是否真的创建了
- 大模型可能"骗你"，说创建了但实际没创建
- 通过 Web UI → 定时任务 → 查看任务列表确认
- 创建方式：聊天创建 / Web UI 新增 / 修改 JSON 配置

### 2. 检查时区
- Cron 表达式必须包含时区
- 确保使用国内时区（Asia/Shanghai），否则可能凌晨4点就执行了

### 3. 配置文件是否禁用了定时任务
- 检查 openclaw.json 中定时任务相关配置是否被禁用

### 4. Gateway 服务是否启动
- 电脑关机或 Gateway 宕机 → 定时任务不执行
- 必须保证服务和电脑正常运行

### 5. 投递渠道配置是否正确
- 检查飞书/Telegram 等渠道的 ID 或 Group ID

## 3个使用技巧

### 技巧1：让 OpenClaw 自己排查
直接问它"为什么之前的定时任务没有执行"，让它诊断并解决。

### 技巧2：使用靠谱的大模型
有些模型会产生幻觉，说创建了实际没创建。

### 技巧3：设置 `forceExecute: true`
在定时任务配置中设置尽可能执行，遇到网络抖动或授权报错仍会执行，只是把错误信息返回。
