## 📌 核心理解

**一句话概括**：一条消息进入系统 → 通道接入 → Gateway 分发 → Agent 拆解与执行 → 工具调用 + 上下文读取 → 生成文件/记录/可复用结果 → 沉淀到 workspace

## 📁 Workspace 详解（03:14）

**Workspace 是 OpenClaw 的工作区**，存储执行过程中产生的内容：

| 内容类型 | 说明 |
|---------|------|
| 文档 | 生成的 markdown、txt 等 |
| 代码 | 生成的脚本、程序 |
| 记录 | 执行日志、对话记录 |
| 任务进度 | 上下文、状态信息 |

## ⚙️ 六大核心配置文件（04:00）

| 文件 | 用途 | 说明 |
|------|------|------|
| **AGENTS.md** | 任务处理定义 | Agent 的行为规则 |
| **SOUL.md** | 人格设定 | 小龙虾的性格、说话风格 |
| **USER.md** | 用户信息 | 主人的资料、偏好 |
| **TOOLS.md** | 工具使用 | 外部工具配置 |
| **MEMORY.md** | 记忆系统 | 长期记忆存储 |
| **HEARTBEAT.md** | 运行状态 | 定时任务、心跳检查 |

## 🎯 关键要点

### 1. 分层架构优势
- 清晰的职责划分
- 易于扩展和维护
- 便于问题排查

### 2. 配置文件关系
```
AGENTS.md  ←→  SOUL.md  ←→  USER.md
     ↓           ↓           ↓
  TOOLS.md  ←→  MEMORY.md  ←→  HEARTBEAT.md
```

### 3. Workspace 的重要性
- 所有执行结果最终沉淀地
- 是 Agent 的"记忆库"
- 支持跨会话持久化
