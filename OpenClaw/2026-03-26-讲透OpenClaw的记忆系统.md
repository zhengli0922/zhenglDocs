# 讲透OpenClaw的记忆系统，原理讲透，模板给全，直接复制就能用

> 来源：今日头条 · 神奇龙虾 · 2026-03-26

## 核心原则

**「说了」不等于「记了」。** 聊天窗口里答应你的事情，关掉窗口就消失了。只有真正写进文件的信息，才算记忆。

聊天窗口 = 便利贴（有大小限制，写满自动撕掉）
记忆文件 = 笔记本（存在硬盘上，关机重启还在）

## 三层记忆结构（优先级从高到低）

| 层级 | 文件 | 用途 | 可靠性 |
|------|------|------|--------|
| 第一层 | USER.md / SOUL.md | 永久偏好、行为规则 | ⭐⭐⭐ 每次启动必读 |
| 第二层 | MEMORY.md | 偏好、重要决定、项目背景 | ⭐⭐ 注意：单文件上限2万字，超出静默截断 |
| 第三层 | memory/YYYY-MM-DD.md | 每日日志流水账 | ⭐ 启动时读取今天和昨天 |

**越重要的事，放的层级越高。**

## 失忆的三个原因

1. **只说没写** — 最常见，聊天里说的没写进文件
2. **上下文压缩** — 对话太长触发压缩，没写进文件的内容直接消失
3. **AI覆写记忆文件** — 用写入工具从头创建MEMORY.md，覆盖原有内容

## 基础修复三件事

### 1. 让AI行动前先搜记忆（加到AGENTS.md）

```markdown
## 记忆规则
- 开始任何任务前，先用 memory_search 搜索相关记忆
- 用户纠正我时，立即将正确信息写入 MEMORY.md
- 用户说「记住这个」时，写入 MEMORY.md 并回复已写入的原文
- 执行不可逆操作前，必须先给用户看执行计划，确认后再动
```

### 2. 禁止AI覆写记忆文件（加到SOUL.md最开头）

```markdown
## 文件操作规则（最高优先级）
- 严禁使用写入工具从头创建 MEMORY.md
- 更新记忆只能使用追加或编辑方式，不能整体替换
- 删除记忆内容前必须获得用户明确确认
- 修改任何记忆文件后，回复我已修改的具体内容
```

### 3. 开启预压缩存盘（memoryFlush）

在 openclaw.json 里配置：
- `memoryFlush.enabled` = true
- `reserveTokensFloor` = 20000
- `softThresholdTokens` = 4000

SOUL.md 兜底规则：
```markdown
## 压缩前存盘规则
- 当对话变长时，主动将重要的决策、指令和偏好写入当天日志文件
- 不要等系统提醒，感觉对话内容较多时就主动存盘
- 存盘后简短告诉我保存了哪些内容
```

## 日常使用模板

### 让它记住
> 把这条写进你的 MEMORY.md：我写作时回复不超过200字，不需要开场白和总结。写完后把你写入的原文发给我确认。

### 让它纠错
> 你之前记的不对。正确的是：[正确信息] 请在 MEMORY.md 里找到原来那条，改成正确版本，并把修改前后的内容都发给我看。

### 总结项目背景
> 请把我们这次对话里关于 [项目名称] 的关键信息整理一下，写进 MEMORY.md，用「## [项目名称]」作为标题。写完发给我确认。

## 常见问题

- **自定义文件名不生效** → 只加载8个固定文件名（SOUL.md, AGENTS.md, USER.md, TOOLS.md, IDENTITY.md, HEARTBEAT.md, BOOTSTRAP.md, MEMORY.md）
- **MEMORY.md被清空** → AI覆写了，加禁止覆写规则
- **长对话后指令消失** → 开启memoryFlush，重要任务前主动存盘
- **记忆越来越不准** → 定期清理MEMORY.md，优先参考近期日志
- **换平台后忘事** → 确认所有渠道使用同一个workspace路径
